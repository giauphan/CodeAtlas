## CodeAtlas MCP — Codebase Intelligence

An MCP server named `codeatlas` is available. It provides analysis data from the CodeAtlas extension, including project structure, dependencies, and code insights.

### When to use CodeAtlas MCP tools:
- **Before manually searching files** — call `get_project_structure` to understand the codebase layout first
- **When looking for a specific function/class** — call `search_entities` instead of grep/find
- **When understanding how files relate** — call `get_dependencies` to see import/call relationships
- **When analyzing code quality** — call `get_insights` for AI-generated insights
- **When exploring a specific file** — call `get_file_entities` to see all entities in that file

### Available tools:
| Tool | Use when |
|------|----------|
| `list_projects` | List all analyzed projects |
| `get_project_structure` | Need overview of modules, classes, functions |
| `get_dependencies` | Need to understand how code connects |
| `get_insights` | Need code quality analysis |
| `search_entities` | Looking for a specific function, class, or module by name |
| `get_file_entities` | Need to know what's inside a specific file |
| `generate_system_flow` | User wants to see/understand system architecture |
| `sync_system_memory` | After completing code changes (ALWAYS call this) |
| `trace_feature_flow` | Before working on a feature (understand context first) |

### Important:
- Data comes from `.codeatlas/analysis.json`, generated by running `CodeAtlas: Analyze Project` in the editor
- If no data found, tell user to run the analysis command first
- Always try CodeAtlas tools BEFORE doing manual file searches — it's faster and gives relationship context
